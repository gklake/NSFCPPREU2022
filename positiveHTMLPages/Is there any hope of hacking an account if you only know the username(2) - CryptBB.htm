I'm going to assume you're trying to gain access to an account on an online website. As with anything, I highly recommend you do recon before you decide what approach you're going to take.

Figure out what software they're using, and specifically what versions. If they're using outdated software (e.g: Apache, Nginx, PHP, Caddy, PHP forum software, etc ...) you may be able to exploit vulnerabilities present.

If you know exactly what software they're using, you can take a look into the code (if the software is source-available like bbforums) of how it handles authentication, such as rate limiting, password hashing, etc. If they use PBKDF2 (with greater than 25,000 iterations), Bcrypt, Scrypt, or Argon2, forget brute-forcing.

Do note that both PBKDF2 and Bcrypt are vulnerable to custom hardware, such as ASICs.

Brute-forcing is generally never the way to go. If you can gain access to the server, that would be preferred. You could modify the source code on their server to store passwords in plaintext in a file, whatever you want. Force all users to be logged out, and then wait for passwords to come in.

If you know a bit about cryptography, you can find specifics ways to exploit it.

Focus on areas where they'll forget about security the most, like a search page, instead of the login page. There is loads of ways to get passwords, and generally it involves no brute-forcing. You may have some luck doing small dictionary attacks.

Best method: pretty much anything BUT brute-forcing. Gain access to the server, modify the code. MitM them if possible. Etc.

I look a lot into a lot of open source projects, and most modern web frameworks incorporate semi-decent to decent rate limiting, and very good password hashing.

You'll have much better luck with older software.
